# Development Roadmap

## Phase 1: 基础壳层

目标：让桌宠先跑起来。

- 初始化 Electron + React + TypeScript 工程
- 创建悬浮宠物窗体
- 创建聊天面板窗体
- 加入系统托盘
- 支持点击展开/收起

验收标准：

- 应用可启动
- 右下角显示桌宠
- 点击可打开聊天窗口

## Phase 2: 智谱对话接入

目标：让桌宠能真正聊天。

- 封装智谱 API 客户端
- 建立主进程 IPC 对话通道
- 支持基础多轮会话
- 本地保存会话记录

验收标准：

- 用户可以发送消息并收到 AI 回复
- 会话历史重启后可恢复

## Phase 3: 文件分析

目标：让桌宠能读文件。

- 支持选择或拖拽文件
- 接入 PDF、DOCX、TXT 解析
- 做摘要和问答
- 将分析结果纳入聊天上下文

验收标准：

- 能对 PDF/DOCX 做摘要
- 能基于文件内容继续追问

## Phase 4: 提醒与待办

目标：让桌宠更像个人助理。

- 新增待办与提醒表单
- 本地调度提醒
- 系统通知弹窗
- 待办完成状态更新

验收标准：

- 到点可收到提醒
- 待办可新增、查看、完成

## Phase 5: 打磨体验

目标：让产品更像“桌宠”。

- 桌宠动效
- 问候语与状态文案
- 更自然的提醒策略
- 设置页
- 开机自启

## 推荐开发顺序

1. 先把壳层和聊天 UI 做出来
2. 接智谱 API，跑通基础聊天
3. 加文档分析
4. 最后补提醒与待办

## 风险点

- Electron 透明窗与点击穿透行为需要仔细调试
- PDF 文本提取质量会受文件格式影响
- 提醒功能需要兼顾系统权限与稳定性
- 如果上下文管理不好，模型成本会上升
